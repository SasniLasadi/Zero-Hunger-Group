Splash screen

Technical Description

This webpage opens with a centered background image which creates an elegant first impression for viewers. The image surface contains a semi-transparent circular masking element which helps users read the text content. The splash screen contains website mission statement with members' names as well as an editorial link button. Through the page editor editors can present comprehensive explanation of the code structure alongside functional details to enable users in understanding website development and maintenance approaches.

Key HTML Tags

  • <html>: element functions as the main starting point for every HTML document which comprises all webpage material.
  • <head>: element contains both document metadata and fundamental information about the page title together with attached external style sheets.
  • <title>: element establishes which title will show up on browser tabs for the webpage.
  • <link>: tag works as a connection for external HTML resources which point to the CSS file named `styles.css`.
  • <meta>: The meta element serves to provide essential metadata which includes trigger automatic redirection through specifying a time duration using the http-equiv attribute.
  • <style>: This style section includes CSS rules directly inside the HTML file to style how the page displays itself.
  • <body>: The body element presents all content along with images and text that will be shown on the webpage.
  • <div>: The <div> element establishes a generic container which allows the grouping of webpage items. This element enables the development of multiple sections which include the splash container and overlay component along with the content material.
  • <h1>: A top-level heading appears through this tag for purposes of displaying the main page title.
  • <p>: Defines a paragraph. The mission statement together with group member details appear inside this section.
  • <strong>: The strong element defines emphasized text which usually appears in bold font. The text contains "Group Members" which appears highlighted through this element.
  • <a>: The tag creates a link which directs users to access `#page-editor.html`.
  • <button>: The button element enables clickable functionality for the Page Editor call-to-action button.
  • @keyframes: This animation equipment inside CSS serves as a rule to establish movement definitions. The animation performs a spinning display through this rule definition.

Key CSS Properties

  • height: 100vh - The height of the body element extends to 100% of viewport height through the property value 100vh.
  • display: flex - The application of flexbox layout enables the body section to arrange its content with enhanced flexibility.
  • justify-content: center - Centers the content horizontally within the flex container.
  • align-items: center - The content in a flex container receives central positioning along its vertical axis.
  • position: absolute - Absolute positioning of elements in the HTML design system functions based on the splash container serving as the nearest positioned relation to the overlay component.
  • background: rgba(147, 145, 145, 0.8) - The application generates a transparent dark gray element with 80% opacity to enhance the legibility of text across the background picture.
  • background-image: url(images/splashscreen.jpg) - The background image of the splash container receives its visual appearance through this command.
  • background-size: cover - The background image receives full coverage of the splash container and maintains its ratio dimensions.
  • text-shadow: 2px 2px 8px rgba(0, 0, 0, 0.9) - The text obtains a bold shadow effect which enhances its contrast and visibility on the background.
  • cursor: pointer - The button changes its cursor to a pointer when the mouse pointer hovers above it.
  • animation: spin 1s linear infinite - The loader element receives an endless rotating animation through this statement.
  • @keyframes spin - The animation for the spinning loader performs an infinite rotation of its element for 360 degrees.

Accessibility

  • Alternative Text for Images: Before using images users with visual impairments should have screen readers present descriptive text to them.
  • Headings Structure: An appropriate heading structure will help users move through content by making navigation easier for screen reader users.
  • Semantic HTML Elements: Using elements like <nav>, <header>, and <footer> ensure that the design include standards that enable assistive technologies to properly decode content then display it through structured accessible methods.

Volunteer Page

Technical Description

Our webpage utilizes interactive volunteer cards to promote various programs which let users involve themselves in hunger relief efforts through the volunteering section. The website experience features two specific web pages for your convenience. The initial web page introduces relevant imagery and creates an important message followed by a booking link for hunger volunteer programs. Users who click on this button will find themselves on the second page where they can learn about detailed volunteer programs along with their descriptions for engagement purposes. users experience effortless navigation which simplifies their ability to find ways to help decrease hunger and take concrete steps toward this goal.

Key HTML Tags

  • <html>: The root element of an HTML document. It contains all the content of the page.
  • <head>: The Document Type Definition section contains metadata details about the document that includes the title and character set information and linked stylesheet and script references. The document metadata section contains information about document titles yet background scripts and character sets in addition to linked stylesheets but refrains from displaying visible webpage content.
  • <title>: The document title defines the word that appears in browser tabs.
  • <body>: The HTML document shows its visible content through this section. The browser shows all textual and visual components as well as displayable elements within this area.
  • <header>: The element functions as either a document header or a section header. Thispage element contains logos and navigation elements as well as introductory content.
  • <nav>: Site navigation menus appear as navigation links in order to assist users with their movement through the website.
  • <ul>: The unordered list element produces list items with bullet pointed display.
  • <script>: Client-side JavaScript is used as defined to add page interactivity. It can be placed either in the <head> or <body> sections. The script block can reside directly inside either the head or body element of an HTML document..
  • <footer>: The footer section functions as a document component that carries contact details in addition to copyright notes or website relations.
  • <a href>: Provides definition of a hyperlink which includes the href attribute to specify the URL destination.
  • <p>:This tag is used to define a paragraph of text.
  • <form>: The code establishes an HTML form with features to obtain information from users. The form can use multiple form components including input fields and buttons and various other form elements.
  • <button type>: This tag is usedd to design a clickable button. The type attribute from this element determines the functional response of this button (you may use it for form submission or standard buttons).

Key CSS Properties

  • background-color: Used to add background colors of elements like body background and banner background.
  • display (Flexbox): This style provides a flexible arrangement system which can apply to elements including the card container and feature items and rows and columns.
  • border-radius: This is used to apply smooth rounded edges to different components such as cards, buttons, tables and containers.
  • box-shadow: Elements receive shadow effects through CSS for creating subtle 3D appearance which appears in cards, buttons and review cards.
  • padding: This feature manages internal spacing which can be used across cards, buttons, text areas and containers.
  • margin: Sets defines element spacing to create gaps that separate cards, buttons and additional components.
  • hover effects:The combination of transform with box-shadow and background-color and other functionalities enables the creation of interactive hover feedback elements (such as cards and buttons).
  • width & height: The size dimensions of elements including images as well as cards and containers become determinable with different measures through this feature.
  • font-size: The program enables users to modify text dimensions for clear reading alongside visual design consistency.
  • color: The color scheme controls all textual elements including headings together with text content and buttons.
  • text-align: This element sets text distribution within containers that affects headings and section labels.
  • max-width: Elements within containers and tables benefit from this rule to regulate their layout through maximum width limitations.
  • overflow: This CSS property controls content overflow behavior for elements where overflow-y: auto creates scrollable content.
  • position: It specifies the placement of components for employing banners or dropdowns while controlling the page layout.
  • @media (Media Queries): Programmers need Sketch for designing adaptable interfaces with elements that resize for various display dimensions.
  • text-transform: The tool gives users control over their text case for creating text that requires uppercase sections.
  • transition: This property enables smooth transitions of background color and transform and box-shadow properties when used in hover effects.

Accessibility

  • Ensure Sufficient Color Contrast: The combination of dark text with a bright background ensures easy text readability.
  • Form Accessibility : All form inputs should include labels using the (<label>) tags along with specific guidance for users to follow.
  • Ordered Lists : screen reader navigation enhances through tagging lists using proper semantic elements like ordered list <ol>and unordered list <ul>.

Content Page

Technical Description

A well-structured layout appears in the content page which utilizes HTML and CSS for its creation. You will find the page segmented through HTML elements which include <header>, <nav>, <main>, <div>, and <footer> for logical section organization. A responsive design that looks attractive emerges from using the CSS properties display: grid; and grid-template-areas;. The user experience benefits from styling features which include background-color; and transition: all 0.5s to improve functionality together with readability. A structured presentation design creates neat logical interfaces which make content delivery more effective.

Key HTML Tags

  • <div>The component functions as a structure to arrange and design content.
  • <nav>This is used to define the navigation menue.
  • <header>This is used to represent the top section of the page.
  • <main>This is used to hold the main content of the web page.
  • <footer>Used to add relevant information to the bottom section of the webpage.

Key CSS Properties

  • display: grid;The system implements format organization by utilizing a grid structure.
  • grid-template-areas; It designs stationary grid sections with specific names.
  • position: sticky;An element remains visible when the viewer scrolls through a page.
  • background-color;Used to set relevant background-color for the elements.
  • transition: all 0.5s;Used to make animation effect more smooth

Accessibility

  • Descriptive links: Instead of using general click here or learn more buttons, descriptive links were added
  • Use Proper Headings (<h1>, <>h2>, etc.) : Using elements like <nav>, <header>, and <footer> to ensure that assistive technologies can interpret and present content in a structured and accessible manner.
  • Alternative text for images: Every image requires a descriptive alt attribute for screen readers to explain it to visually impaired users. Visual impairments users benefit from screen readers who utilize alt attributes to explain images to them.

Challenges and Lessons Learned

  1. Responsiveness Issues: The page needed additional media queries in its CSS to develop full device responsiveness.Media queries were essential to guarantee desirable user experiences across the different device platforms.
  2. Interactive Elements: The addition of hover effects together with clickable buttons to volunteer cards made the interface more complex. The web application faced difficulties when making sure interactive features operated perfectly between different web browsers during examination.
  3. Testing and Debugging: Testing and debugging the entire feature set demanded most of our time as we checked both JavaScript execution and the prevention of layout shifts and broken links throughout the development process.
  4. Focus on Simplicity: You should arrange the layout into an organized and simple design that helps users navigate towards volunteering information easily.
  5. Responsive Design: Make the page attractive along with proper functionality across all devices especially smartphones to enable seamless user experience at any screen resolution.
  6. Clear Call to Action:One must place the "Volunteer to End Hunger" button at the forefront to encourage direct and instant user participation.

Compliance

The web pages Splash Screen, Volunteer Page, Page Editor and Content Page are built in line with Janet regulations to guarantee security and accessibility and manage content properly. Semantic HTML elements including <header>, <main>, and <footer> improve content accessibility since assistive technologies find it simpler to understand the HTML structure. Images gain accessibility for users with visual impairments through alt attributes as well as appropriate font sizes together with proper color contrasts which enhance content readability. A user-friendly interface connects different webpages which allows users to retrieve relevant content without losing their interest.

The websites achieve user friendliness through compliance with Janet regulations delivering accessibility along with security and ethical compliance standards. Users can efficiently access the website through standard web practices which produce security and compliance benefits for their data privacy alongside ethical professional content delivery.
