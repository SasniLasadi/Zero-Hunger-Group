Key CSS Properties

  1. Layout
    2.

    display:flex; is used in sections like .container, .about and .purpose to create flexible layouts.

    flex-wrap; is used to make the layout responsive by wrapping the items onto multiple lines.

    justify-content: space-evenly; evenly spaces out items having equal spacing between the items in .about and .purpose sections.

  2. Positioning and Layout Control
    3.

    position: absolute; is used in .text and .article to position them relative to their closest parent element.

    transform: translateY(); creates smooth transitions when .background and .card sections hover over.

    width: fit-content; is used in .text class to make the width ajust to content inside.

  3. Visual Appearance
    4.

    background: rgba(0, 0, 0, 0.5); adds a background with an opacity of 0.5 in .text and .article sections.

    color:white; applies for the texts white color.

  4. Transitions and Hover Effects
    5.

    transition: 0.5s ease-in-out; adds smooth transitions for transform and opacity properties.

    :hover is used in .backgroun:hover, etc to change the appearence when the user hovers on.

  5. Sizing and Spacing
    6.

    Width; and height; decides the width and height of .card, .task, etc.

    padding; adds internal spacing in .text section.

    margin; adds spacing between the cards in .card section.

  6. Button Styling
    7.

    background-color: rgb(131, 230, 18); indicates the background colour of buttons.

    cursor: pointer; Changes the cursor to a hand symbol when hovering over the buttons.

Challenges and Lessons Learned

  1. Home Page
    2.

    The site experienced loading delays because of the oversized images displayed on the main page. Optimization of images through the paint tool allowed me to lower their file size without compromising visual quality. Merging max-width: 100% properties with the images allows them to scale automatically based on the current layout dimensions. I set image height at 400px using the property in order to create uniform image dimensions no matter the device screen size.

  2. Table Page
    3.

    The table page featured country flags together with other content while I encountered inconsistent flag display according to screen dimensions. The country flags appeared either improperly aligned or cut off from their original dimensions. I confined every flag image through a restricted container with specific dimensions (height: 30px; width: 40px;). I implemented object-fit: contain; while constraining each image container to handle proper resizing without any image distortion. I resolved issues with mobile layouts using media queries that changed the table rows into vertical positions to improve flag image scaling.

  3. Content Page
    4.

    I struggled to make my content including the sidebar and main content effectively resize properly for smaller screens during the process of developing screen size layouts. The CSS Grid Layout together with Flexbox properties allowed me to handle responsive design needs. The combination of grid-template-columns: repeat(2, 1fr) and grid-template-areas allowed design modifications across various screens to become simpler. Media query application allowed appropriate resizing of the layout which maintained proper stacking of sidebar content and main content elements on mobile screens.
